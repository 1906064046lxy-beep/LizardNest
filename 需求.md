# 蜥蜴巢穴配套 App 需求文档（MVP）

## 1. 项目概述

### 1.1 产品简介
蜥蜴巢穴配套 App 是一款面向蜥蜴巢穴智能硬件用户的 Android 应用。用户可通过 App 实时查看巢穴内的温湿度数据及摄像头直播画面，实现对宠物蜥蜴生活环境的远程监控。

### 1.2 目标用户
购买了蜥蜴巢穴智能硬件产品的用户。

### 1.3 MVP 范围
本版本为最小可行产品（MVP），仅包含核心监控功能。账号系统、报警通知、设备配网等功能不在本版本范围内。

---

## 2. 功能需求

### 2.1 温湿度监控

#### 2.1.1 实时数值显示
- 在首页展示当前巢穴内的**温度**和**湿度**实时数值。
- 温度单位：摄氏度（℃），保留一位小数。
- 湿度单位：百分比（%），保留一位小数。
- 数据刷新频率：待定（建议 5-10 秒自动刷新）。

#### 2.1.2 24 小时历史曲线
- 以折线图形式展示最近 24 小时的温度变化曲线。
- 以折线图形式展示最近 24 小时的湿度变化曲线。
- 两条曲线可在同一图表中展示（双 Y 轴），或分两个图表展示（待定）。
- 图表支持横向滑动查看不同时间点数值。
- X 轴：时间（24 小时范围）。
- Y 轴：温湿度数值。

### 2.2 直播画面

#### 2.2.1 实时视频播放
- 在首页展示巢穴内摄像头的实时直播画面。
- 纯观看模式，不含截图、录屏、双向语音等功能。
- 画面延迟：3-5 秒以内。
- 支持横屏全屏观看。

#### 2.2.2 视频播放控制
- 播放/暂停。
- 横屏/竖屏切换。

### 2.3 页面布局

#### 2.3.1 首页布局（方案 A：单页综合视图）
首页在一个页面内展示所有核心内容，自上而下分为三个区域：

```
┌─────────────────────────┐
│       直播画面区域        │  ← 约占屏幕 50%-60%
│     （摄像头实时画面）      │
│                         │
├─────────────────────────┤
│  温度：28.5℃            │  ← 实时数值
│  湿度：65.2%            │
├─────────────────────────┤
│    24h 温湿度曲线图       │  ← 可左右滑动
│   📈                   │
│                         │
└─────────────────────────┘
```

- **直播画面区**：位于页面上方，默认 16:9 比例展示，支持点击进入全屏。
- **实时数值区**：直播画面下方，清晰展示当前温度和湿度。
- **历史曲线区**：页面底部，展示过去 24 小时的温湿度折线图，支持手势滑动查看。

## 3. 技术需求

### 3.1 技术栈
| 项目         | 选型                        |
| ------------ | --------------------------- |
| 开发语言     | Kotlin                      |
| UI 框架      | Jetpack Compose             |
| 最低支持版本 | Android 8.0（API 26）       |
| 目标版本     | Android 14（API 34）        |
| 构建工具     | Gradle + Kotlin DSL         |
| 网络请求     | OkHttp + Retrofit           |
| 视频播放     | ExoPlayer（Media3）         |
| 图表         | MPAndroidChart 或 Compose 原生 Canvas 绘制 |
| 架构模式     | MVVM                        |

### 3.2 硬件通信抽象层
由于硬件型号暂未确定，所有与硬件的通信（温湿度数据获取、视频流获取）均通过**统一接口层**实现。MVP 阶段使用 **Mock 模拟数据**驱动开发和调试，待硬件确定后切换为真实实现。

#### 3.2.1 数据接口抽象
```kotlin
// 温湿度数据源接口
interface EnvironmentDataSource {
    suspend fun getCurrentData(): TemperatureHumidity  // 当前温湿度
    suspend fun getHistoryData(hours: Int): List<TemperatureHumidity>  // 历史数据
}

// 视频流数据源接口
interface VideoStreamDataSource {
    fun getStreamUrl(): String  // 返回视频流 URL
}
```

#### 3.2.2 MVP Mock 实现
- 温湿度数据：随机生成合理范围内的模拟数据（温度 20-35℃，湿度 40-80%），定时更新。
- 视频流：使用本地占位画面或公开测试流。

### 3.3 远程访问
- 用户需能在离家状态下通过互联网访问巢穴数据。
- MVP 阶段可通过直连 IP/域名方式访问，具体方案待硬件通信方式确定后细化。

## 4. 非功能需求

### 4.1 性能
- App 启动时间不超过 2 秒（冷启动）。
- 直播画面首帧显示时间不超过 3 秒。
- 温湿度数值刷新流畅，无卡顿感。

### 4.2 稳定性
- 网络断开时，界面显示明确的断连提示，不崩溃。
- 视频流中断时自动重连。
- 24 小时内不应出现崩溃（Crash Free Rate > 99%）。

### 4.3 兼容性
- 支持 Android 8.0 及以上系统。
- 适配主流分辨率（1080p、1440p 等）。
- 适配刘海屏、挖孔屏。

### 4.4 安全性
- 视频流和传感器数据传输建议加密（具体方案待硬件确定）。
